Manual Restore

Restore only while HolyFHIR is closed. The backup is encrypted with the same local key. Save a HolyFHIR Recovery Kit and keep it separate from your database backup when possible.

  1. Close HolyFHIR completely.
  2. Open the app data folder shown below.
  3. Rename the current live database to keep a last-chance copy.
  4. Copy the backup file you want from the backups folder.
  5. Rename the copied backup to the live database filename.
  6. If HolyFHIR says the database key is missing, restore it from your Recovery Kit.
  7. Reopen HolyFHIR.

Restore Key Command

If HolyFHIR cannot open your restored database, use this command with your saved Recovery Kit: 

python manage.py import_recovery_kit --recovery-kit-file HolyFHIR-Recovery-Kit.txt --credential-storage system

Example

Rename the live database:

holyfhir.encrypted.sqlite3
holyfhir.encrypted.sqlite3.before-restore

Then copy a backup into the app data folder and rename it:

holyfhir.encrypted.pre-fhir-import.20260426-120000.sqlite3
holyfhir.encrypted.sqlite3
